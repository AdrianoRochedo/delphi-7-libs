Tecnologia

TPascalScript foi rigorosamente projetado nos moldes da Orientação a Objetos (OOP) e escrito em Borland Delphi 5.

Para a compilação foi escolhido o método Descendente Recursivo Preditivo.

Etapas do processo de compilação

A primeira etapa é a Análise Léxica que tem o objetivo de quebrar o texto do código em pedaços chamados de "Tokens".

Neste projeto o responsável por realizar a Análise Léxica é a Classe TPascalScanner que encapsula um Autômato Finito.

Após esta etapa os tokens são acessados pelo compilador da classe TPascalScript para que seja realizado a Análise semântica e ao mesmo tempo a geração de código.

Se tudo correr bem, isto é, o script não contiver erros, o mecanismo da classe TPascalScript estará apto a executar o código gerado. Este código fica armazenado em uma instância da classe TByteCode que é acessada pela Máquina Virtual, aqui representado pela classe TVirtualMachine que tem como atributo principal interpretar o código gerado.

Processo de Execução

Com o p-code gerado e acessado pela Máquina Virtual basta interpretá-lo e transformar os p-codes em comandos válidos para o sistema operacional em uso.

O mecanismo mais complicado do processo de execução é a ligação do p-code com as rotinas criadas pelo usuário. Neste mecanismo optei por passar os parâmetros das rotinas através de uma pilha a qual pode ser acessada nos métodos de acesso as rotinas do usuário.

Ex: o usuário acessaria o segundo parâmetro de uma função através dos possíveis métodos da classe TexeStack, por exemplo Stack.AsString(2) e devolveria o resultado da função através dos métodos PushXXX(Valor) onde XXX é o tipo do dado. Ex: Stack.PushFloat(3.3) ou Stack.PushString('Sucesso').

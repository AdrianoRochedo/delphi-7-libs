Elementos Sintáticos
O Pascal Script utiliza o conjunto de caracteres ASCII, incluíndo as letras de "A" a "Z" e "a" a "z", os dígitos de "0" a "9" e outros caracteres padrão. Ele ignora a diferença entre letras maiúsculas e minúsculas.

Os Elementos Sintáticos Fundamentais, chamados Tokens, combinam-se para formar expressões, declarações e comandos. Um comando descreve uma ação algoritmica que pode ser executada com um programa.

Uma expressão é uma unidade sintática que ocorre com um comando e denota um valor. Uma declaração define um identificador (tal como um nome de função ou variável) que pode ser utilizado em em expressões e comandos, e, onde for apropriado, alocar memória para um identificador.

Elementos Sintáticos Fundamentais
Em um nível simples, um programa é uma sequência de tokens delimitados por separadores. Um token é a menor unidade de texto em um programa. Um separador é um espaço em branco ou um comentário. Estritamente falando, não é sempre necessário colocar um separador entre dois tokens; por exemplo, o fragmento de código

Size:=20;Price:=10;

é perfeitamente legal. Convensão e legibilidade, no entanto, ditam que nós deveríamos ter escrito

Size := 20; Price := 10;

Tokens são categorizados como símbolos especiais, identificadores, palavras reservadas, números e caracteres strings. Um separador pode ser parte de um token somente somente se o token é um caracter string. Identificadores adjacentes, palavras reservadas e números devem possuir pelo menos um separador entre eles.

Comentários
Comentários säo textos introduzimos no meio do código do programa com a intençäo de torná-lo mais claro. É uma boa prática em programaçäo inserir comentários no meio dos nossos programas. No Script, tudo que estiver entre os símbolos "{" e "}" ou após o símbolo "//" será considerado como comentário.

Exemplos: 
  // Isto é um comentário de uma linha

  {
  Isto é um bloco
  com duas linhas de comentário
  }
Estrutura de um programa em PascalScript
Todo programa em PascalScript é subdividido em 3 áreas: 

        - cabeçalho do programa [Opcional]
        - área de declaraçöes [Opcional]
        - corpo do programa
Na definiçäo padräo da linguagem Pascal, o Cabeçalho do programa é obrigatório, no entanto, no Script ele é opcional.

A área de declaraçöes é o lugar onde são definidas as variáveis.

O PascalScript possui 5 tipos de variáveis pré-definidas a saber: Integer, Real, Boolean, String e Object.

Todas as variáveis utilizadas no programa devem ser declaradas na sub-área Var, pois a alocaçäo de espaço de memória para as variáveis é feita durante a compilaçäo.

Identificadores
O PascalScript possui inúmeros identificadores pré-definidos, que näo fazem parte da definiçäo padräo da linguagem Script Pascal. Esses identificadores consistem em Procedures e Functions, que podem ser utilizados normalmente na construçäo de programas. Exemplos: 

    
    Beep          : Emite um som
    ShowMessage() : Mostra uma mensagemna tela
Regras para formaçäo de identificadores:

    1-) O primeiro caractere do identificador deverá ser obrigatoriamente uma letra ou um underscore "_".
    2-) Os demais caracteres podem ser letras, dígitos ou underscores.
    3-) Näo pode ser palavra reservada.
Exemplos de identificadores válidos: 

        Meu_Nome
        MEU_NOME   (igual ao anterior)
        __Linha
        EXemplo23
Exemplos de identificadores näo válidos: 

        2teste      : começa com número
        Exemplo 23  : tem um espaço
Identificadores de Qualificação

Qualificadores podem ser iterativos.
Por exemplo: OutPut.Show
chama o método Show do objeto OutPut

Números
No Pascal Script, podemos trabalhar com números inteiros e reais. Os números reais também podem ser representados em notação científica.

Números inteiros são representados internamente como números reais.

Characteres String
Strings säo conjunto de caracteres entre aspas simples. Exemplos: 

  'isto é uma string' { isto é uma string }
  '123456'            { 123456 }
  ''''                { ' }
  ''                  { string nula }
  ' '                 { um espaço }
Sobre Expressões
Uma expressão é um conjunto de operandos unidos por operadores, de forma a computar um valor ou resultado.

Os operandos podem ser variáveis, constantes ou valores gerados por funções.

Os operadores identificam as operações a serem efetuadas sobre os operandos. Cada tipo de dados possui um conjunto de operadores relacionados. Os operadores classificam-se como Unários e Binários, conforme tenham um ou dois operandos, respectivamente. 

  X                   { variável }
  15                  { constante inteira }
  InterestRate        { variável }
  Calc(X,Y)           { chamada de função }
  X * Y               { produto de X por Y }
  Z / (1 - Z)         { divizão de Z por (1 - Z) }
  X = 1.5             { Lógico (Boolean) }
  not Feito           { negação de uma variável lógica (Boolean) }
Sobre Operadores
Operadores tais como as funções pré-definidas são partes da linguagem Pascal Script.

Operadores incluem:
   *, /, +, –, mod, div, and, or, not, =, >, <, <>, <=, >=

Operadores Aritiméticos
 
    +     adiçäo
    -     subtraçäo
    *     multiplicaçäo
    /     divisão real
    div   divisão inteira
    mod   resto de uma divisão inteira

Exemplo

    program Operadores_aritimeticos;
    var x, y, z : integer;
        r1, r2  : real;
    begin
       x := 10;
       y := 20 div 10;
       z := x+y mod y;
       OutPut.Write(z); // escreve o valor de z em um console
       x := 20 / 3;
       y := 20 * 3;
       OutPut.Write(x);   
       OutPut.Write(y);   
       r1 := 3.24;
       r2 := r1 / 2.3;
       OutPut.Write(r2);
    end.
Operadores Lógicos (Booleanos) 

    AND   -->  E lógico
    OR    -->  OU lógico
Estes operadores só aceitam como operandos, valores lógicos, ou seja, valores TRUE ou FALSE .

A operaçäo AND resulta em TRUE se e somente se todos os operandos forem TRUE, se um deles ou mais de um for FALSE então o resultado será FALSE.

A operaçäo OR resulta TRUE quando pelo menos um dos operandos for TRUE.

Exemplo 1: 

    program operadores_logicos;
    var x,y : boolean;
    begin
      x := TRUE;
      y := FALSE;
	   
      if x OR y then
         OutPut.Write('Esta linha será impressa');
		  
      if x AND y then  
         OutPut.Write('Esta linha não será vista');
    end.

Exemplo 2:

    while (I <= Length(S)) and (S[I] <> ',') do
      begin
      ...
      Inc(I);
      end;
Operadores String
Os operadores relacionais =, <>, <, >, <=, and >= operam também sobre strings. O operador "+" concatena duas strings. 

Exemplo 1:
    b := ('Adriano' = 'adriano') --> FALSO

Exemplo 2:
    'Isto é uma ' + 'string' --> 'Isto é uma string'


Operadores Relacionais
O Pascal Script possui ao todo 6 operadores relacionais que säo muito utilizados nas tomadas de decisöes, säo eles: 

    =       igual
    <>      diferente
    >       maior que
    <       menor que
    >=      maior ou igual que
    <=      menor ou igual que

    Exemplos:

    1) Se A=30  e B=50 entäo
         ( A = B ) --> FALSE
         ( A < B ) --> TRUE

    2) Se A=TRUE e B=FALSE então
         ( A <> B ) --> TRUE
         ( A =  B ) --> FALSE

    3) Se A=50 , B=35, C='A' , D='B' então
         ( ( A < B ) OR ( C < D ) ) --> TRUE
A avaliaçäo será verdadeira se uma ou outra expressão for verdadeira, no caso, como C < D entäo a resposta é TRUE

Precedência dos Operadores
Se vários operadores aparecerem em uma expressão, a ordem de execução das operações será dada segundo os critérios abaixo :

  • pelo emprego explícito de parênteses
  • pela ordem de precedência existente entre os operadores
  • se existirem operadores de mesma ordem de precedência, a avaliação será feita da esquerda para a direita.
Ordem de Precedência dos Operadores ( da maior para a menor ) 

  Operadores                    Categoria
  (  )                          Parêntesis
  @   not                       Operadores Unários
  *  /  div  mod                Operadores Multiplicativos
  and  +  -  or                 Operadores Aditivos
  =  <>  <   >  <=   >=         Operadores Relacionais
Exemplos 

X + Y * Z
Multiplica Y vezes Z, então adiciona X ao resultado; "*" é realizado primeiro, porque sua prioridade é maior do que a do "+". Mas 

X - Y + Z
primeiro subtrai Y de X, então adiciona Z ao resultado; "-" e "+" possuem a mesma precedencia, então a operação da esquerda é realizada primeiro.

Você pode usar parenteses para controlar as regras de precedencia. Uma expressão com parênteses é avaliada primeiro, eles são tratados como operandos simples

Exemplo 

(X + Y) * Z
multiplica Z vezes a soma de X por Y.

Parenteses são as vezes necessários em situações onde, a primeira vista, não seriam necessários. Por exemplo, considere a expressão: 

X = Y or X = Z
A interpretação lógica desta expressão é obvia 

(X = Y) or (X = Z)
Sem parênteses, no entanto, o compilador seguirá as regras de precedencia e lerá 

(X = (Y or X)) = Z
no qual o resultado é um erro de compilação a menos que Z seja um Booleano.

Parenteses oferecem facilidade para se escreer e ler código, já que, eles são, estritamente falando, superfluos. Repare na expressão abaixo: 

X + (Y * Z)
Aqui os parenteses não são necessários (para o compilador), mas eles tornam o código claro para o programador enfatizando a precedencia.

Chamada de Funções
Por que funções retornam um valor, funções podem ser chamadas em expressões. Por exemplo, se você definir uma função chamada "Calc" com dois argumentos e de retorno inteiro, então a chamada de Calc(22, 45) é uma expressão inteira.

Exemplos de chamada de funções 

  Sum(A, 63)
  Max(147, J)
  Sin(X + Y) 
  Volume(Raio, Largura)
  GetValue
  AlgumObjeto.AlgumMetodo(X, Y, Z)
Typecast
Serve para tratar um tipo de objeto como outro.
A sintáxe para o typecast é: 

    tipoIdentificador(variável Object)
Exemplo: 
     program Planeja_LiberaDemandas_PC9;
     var V  : Object;
         dt : Integer;
     begin
       V := TwsVec(GlobalObjects.Get('Meu vetor'));
       if ObjectIsValid(V) then V.Print(Saida);

Sobre Declarações e Comandos
Um programa consiste enteiramente de declarações e comandos, os quais são organizados em blocos.

Constituição básica de um programa 

  - Identificação do Programa [opcional]
  - Bloco de Declarações de Variáveis [opcional]
  - Bloco Principal
    - Comandos Simples ou Compostos
Ex: 

    program Identificacao;
    var
       v1, v2: Tipo;
       v3, v4: Tipo;
    begin
       comando 1;
       comando 2;
       ...
       comando N;
    end.
Declarações
Uma variável é um valor que pode ser alterado dentro de um programa. Todas as variáveis usadas num programa devem ser declaradas usando a cláusula VAR, dentro da seção de declarações. Nesse comando, além de indicar o nome que deseja para as variáveis, indica-se também qual o tipo que as mesmas estão associadas.

Sintaxe: 

var   identif [ , identif ]... : tipo-do-dado;
    [ identif [ , identif ]... : tipo-do-dado; ] ...
O tipo-do-dado é um dos tipos pré-definidos.

Ex.: 

  var 
    Soma, Total, Salario : Real;
    Idade, Contador      : Integer;
    Planilha             : Object;
    Nome                 : String;
    PossuiDireitoFerias  : Boolean;
Tipos de dados pré-definidos 

   - Real        Números em Ponto Flutuante 
   - Integer     Números Inteiros
   - Boolean     Valores Lógicos (true/false)
   - String      Sequência de Caracteres entre Aspas
   - Object      Instâncias de Classes
Comandos
Comandos definem ações algorítmicas em um programa. Comandos simples como atribuições e chamadas de procedimentos podem se combinar para formar laços, comandos condicionais e outros comandos estruturados.
Multiplos comandos em um bloco devem ser separados por um ponto e vírgula.

Comandos Simples
Um comando simples não contém qualquer outro comando. Comandos simples incluem atribuições, chamadas de procedimentos e funções.

Comando de Atribuição
Usado para atribuir o valor de uma expressão a uma variável.

Sintaxe:
     Variável := Expressão
Exemplos: 

     A := 10;
     Nome := 'Adriano Rochedo';
     X := Y + Z;
     Terminado := (I >= 1) and (I < 100);
     I := Sqr(J) - I * K;
     Planilha := CreateObject('TPlanilha');
     X := umVetor.get(5) * umVetor.get(6);
Chamada de Funções e Procedimentos
Uma chamada de procedimento consiste em uma referência a um nome de procedimento seguido de sua lista de parâmetros se requerido. Um procedimento pode pertencer a uma classe, neste caso ele faz parte dos métodos desta classe.

Exemplos: 

  PrintHeading;
  Transpose(A, N, M);
  Find(Smith, William);
  OutPut.Write('Hello world!');
  DoSomething(.., .., ..);
  MeuObjeto.AlgumMetodo(X, Y);
Funções, tais como os procedimentos, podem ser tratadas como comandos. 

  MinhaFunção(X);
Quando você usa uma função deta maneira, seu valor de retorno é discartado.

Comandos Estruturados
Comandos estruturados são construídos com base em outros comandos. Utilize um comando estruturado quando você querer executar outros comandos sequencialmente, condicionalmente ou repetidamente.

Um comando estruturado pode ser formado por um ou mais blocos constituíntes.

Um comando condicional, isto é, um comando IF executa um dos seus dois blocos constituintes dependendo do critério especificado.

Comandos que formam laços, tais como os comandos WHILE e FOR, executam uma sequência de comandos repetidamente.

Comandos Compostos
Consiste de um ou vários comandos que devam ser executados sequencialmente, sendo usado em situações onde a sintaxe de alguns comandos do Pascal permite apenas um único comando.

Os comandos que fazem parte de um comando composto são separados por ponto-e-vírgula, devendo começar e terminar pelas palavras reservadas BEGIN e END respectivamente, de acordo com a sintaxe: 

    begin
    comando [ ; comando ]...
    end;
Exemplo: 

    if A > B then
       begin
       Temp := A;
       A    := B;
       B    := Temp;
       end;
Comando IF
O comando IF permite ao programa tomar decisões e pode ter duas sintaxes:

Primeira sintaxe: 

    if expressão_lógica then Comando;
Se a expressão_lógica resultar em um valor verdadeiro (TRUE), entäo o comando será executado caso contrário não. Para os casos em que tivermos mais de um comando para serem executados, eles deveräo ser acomodados em um bloco "Begin End" . 

    if expressão_lógica then
       begin
       Comando_1;
       Comando_2;
       Comando_N;
       end;
No caso acima, se expressão_lógica for TRUE entäo todos os comandos inclusos entre as palavras "begin" e "end" seräo executados, caso contrário näo.

Segunda sintaxe: 

    if expressão_lógica then 
       Comando_1
    else 
       Comando_2;
Neste caso, se expressão_lógica for TRUE entäo comando_1 será executado e comando_2 não, caso contrário, comando_2 será executado e comando_1 não. Repare que näo temos ";" no final de comando_1 pois o comando ainda não está terminado.

Podemos também escrever: 

    if expressão_lógica then 
       begin
       Comando_1;
       Comando_2;
       end // não tem ";"
    else 
       begin
       Comando_3;
       Comando_4;
       end;
Exemplos: 

    program Exemplo_1;
    var i: Integer;
    begin
      i := InputBox_i('Digite um inteiro maior que 100');
      if i > 100 then
         ShowMessage('Maior que 100 !!')
      else 
         ShowMessage('Menor ou igual a 100');
    end.
	
    program Exemplo_2;
    var n1, n2: Integer;
    begin
      n1 := InputBox_i('Primeiro número');
      n2 := InputBox_i('Segundo número');
      if n1 > n2 then
         ShowMessage('O primeiro número é o maior')
      else 
         if n2 > n1 then
            ShowMessage('O segundo número é o maior')
         else 
            ShowMessage('Os dois números são iguais');
    end.
Controle de Loops
Laços permitem que o programador execute um sequência de comandos repetidamente, usando uma condição de controle ou variável para determinar quando a execução irá terminar.

No PascalScript possuímos dois tipos de Laços: While e For

Laço While
A estrutura While..Do permite controlar o número de vezes que um comando ou um bloco de comandos será executado.

Sintaxe 

    while expressão_lógica do comando;

    ou

    while expressão_lógica do
      begin
      comando_1;
      comando_2;
      end;
Exemplo: 

    program Exemplo_1;
    var i: Integer;
    begin
      OutPut.Clear;
      i := 0;
      while (i < 11) do
        begin
        OutPut.WriteInteger(i);
        i := i + 1;
        end;
    end.
Laço For
Este comando permite que comandos sejam repetidos um certo número de vezes.

Sintaxe 

    for [variável] := [valor_inicial] to/downto [valor_final] 
	[step incremento] do comando;
"step" assume o valor 1 quando omitido.

A variável deverá ser, obrigatoriamente, do tipo Integer ou Real. A variaçäo da variável entre valor inicial e valor final será crescente e de "step" em "step", quando utilizamos a palavra "to" e decrescente de "step" em "step", quando utilizamos a palavra "downto".

Exemplos: 

    program Exemplos_de_For;
    var i: Integer;
	    r: Real;
    begin
       OutPut.Clear;

       for i := 10 to 15 do 
         OutPut.WriteInteger(i); 

       for i := 10 to 150 step 10 do 
         OutPut.WriteInteger(i); 
		 
       for r := 0.1 to 1 step 0.1 do 
         OutPut.WriteFloat(r); 
		 
       OutPut.Show;	 
    end.

    Program Fatorial; 
    {
    Programa para cálculo do fatorial de um número. 
    Lembrando que o fatorial de um número é definido
    como segue:
       N! = 1 * 2 * 3 * 4 * ... * N
       e  
       0! = 1
    }
    Var n, Fatorial, i: Integer;
        msg: String;
    begin
      n := InputBox_i('Entre com um número');
      if n >= 0 then 
         begin
         Fatorial := 1;
         
         for i := 1 to n do
           Fatorial := Fatorial * i;
		
         msg := 'Fatorial de ' + ValToStr(n) + ' ' +
                '= ' + ValToStr(Fatorial);
         ShowMessage(msg);
         end;
    end.
Criação de Objetos
O Pascal Script é uma linguagem baseada em objetos, e como tal, permite a criação de instâncias de classes.

A maior parte dos objetos podem ser criados através do comando CreateObject(). Outros só podem ser criados se existirem funções específicas.

Comando CreateObject
Este comando permite a criação de instâncias de classes de objetod.

Sintaxe: 
    CreateObject('Nome da Classe')
Exemplo: 
    Program Teste_Panilha;
    Var
       i, ii  : Integer;
       S      : String;
       P      : Object;
    begin
      P := CreateObject('TPlanilha');
    
      P.SetCellFont(1, 1, 'ARIAL', 14, 0, True, True, False);
      P.Write(1, 1, 'Adriano');
    
      for i := 5 to 10 do 
        P.WriteFloatCenter(i, 3, i*0.3);
		
      P.Copy;
      P.SaveToFile('D:\temp\ScriptPlanilha.xls', 'XLS');
      P.SaveToFile('D:\temp\ScriptPlanilha.txt', 'TXT');
      P.Show;
    end.
